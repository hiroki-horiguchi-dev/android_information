# 2部 読み方

## プログラミング書の読み方
- 作るものの目的を理解
- 複数言語を比べながら
- 動くものを作る
  - エラーを恐れない
  - 別解をいくつか考えてみる
- 自分の手で作る
  - 他人のコードを信用しない


## 数学書の読み方
- 必要な時に必要な分野のみ学びなおす
- 理解に時間をかける
